Embedding Panel Apps

This page demonstrates how Panel apps can be embedded in any HTML page with minimal boilerplate. Each approach uses a shared Pyodide runtime via panel-runner.html in an iframe.

1. Simple App Embed (No Editor)

Embed a Panel app by providing Python code. The iframe loads Pyodide once and renders the app. Users interact with the running app but cannot edit the code.

Embedding is as simple as:

<script type="panel" data-iframe>
import panel as pn
pn.Column("# Hello!", "Panel runs in the browser!").servable()
</script>

2. Inline Editor + App

A compact playground widget: editable code on top, live app below. Users can modify the code and click Run to see changes.

3. Auto-Servable (Just Return a Value)

If the code doesn't call .servable(), the playground automatically wraps the last expression with pn.panel(). This makes embedding even simpler for quick demos.

4. Code Below Layout (iframe)

Using data-layout="code-below" with iframe mode.

5. Collapsible Code (iframe)

Using data-code="hidden" with iframe mode. The app renders immediately; click <> Code to view source.

6. External Source (iframe)

Load Python code from a URL using src with iframe mode.

7. The Dream: Minimal Boilerplate

The <script type="panel"> tag is all you need! The browser ignores unknown script types, so the Python code is inert until panel-embed.js processes it on page load. 

<script type="panel">
import panel as pn
slider = pn.widgets.FloatSlider(name="Value", start=0, end=10)
pn.Column(slider, pn.bind(lambda v: f"Value: {v}", slider)).servable()
</script>

<!-- With editor -->
<script type="panel-editor" data-title="My App">
import panel as pn
pn.Column("# Hello", "World").servable()
</script>

<!-- Code below with collapsible toggle -->
<script type="panel-editor" data-code="hidden" data-layout="code-below">
import panel as pn
pn.Column("# Hello").servable()
</script>

<!-- Load from external file -->
<script type="panel" src="../examples/hello.py"></script>
