Panel + PyScript POC: Editor App

This demonstrates PyScript's <py-editor> component for editable Panel code. Users can modify the code and press the Run button (or Ctrl+Enter) to re-execute.
Known limitation: py-editor runs code in a web worker, which is isolated from the main thread. Panel's .servable() relies on main-thread DOM access and Bokeh's rendering pipeline. This means py-editor cannot directly render Panel widgets via .servable(). Below we show two approaches: one using py-editor with pyscript.display() for simple output, and one using a main-thread script for full Panel rendering.

Approach 1: py-editor (Worker) - Text Output Only

This editor runs in a worker. It can compute and print results, but cannot render interactive Panel widgets because the worker has no direct DOM access for Bokeh's rendering pipeline.

Approach 2: Main-Thread Rendering (Full Panel Support)

For full Panel interactivity, we run on the main thread. This approach uses a regular <script type="py"> tag. The code is not editable via py-editor, but it demonstrates that Panel rendering works correctly with PyScript's main-thread execution.

Since py-editor runs in a worker (incompatible with Panel rendering), we can build a simple editor using a textarea that sends code to PyScript's main-thread interpreter. This is what panel-embed.js does with CodeMirror, but here we demonstrate the minimal viable version.
