Embedding Panel Apps (Single Pyodide)

This page demonstrates the same demos as embed.html, but uses a single Pyodide instance instead of 3 separate iframes. This reduces memory from ~1.5GB to ~300-500MB, avoiding browser crashes.

Each app renders into its own <div> container using Panel's write() / write_doc() with scoped DOM queries and isolated exec namespaces.

1. Simple App Embed (No Editor)

Embed a Panel app by providing Python code. The single Pyodide runtime renders the app into a target div. Users interact with the running app but cannot edit the code.

2. Inline Editor + App

A compact playground widget: editable code on top, live app below. Users can modify the code and click Run to see changes.

3. Auto-Servable (Just Return a Value)

If the code doesn't call .servable(), the runtime automatically wraps the last expression with pn.panel(). This makes embedding even simpler for quick demos.

4. Code Below Layout

Using data-layout="code-below", the preview appears above the editor. Great for documentation where you want the result to be the first thing readers see.

5. Collapsible Code (Hidden by Default)

Using data-code="hidden", the code section starts collapsed. Click the <> Code button to expand it. This is the MUI-style pattern where the app is the primary focus.

6. Collapsible + Code Below

Combining data-code="hidden" with data-layout="code-below". The toggle button sits between the preview and the collapsible code section below.

7. External Source (src attribute)

Load Python code from a URL using the src attribute. Works with both <script type="panel"> and <script type="panel-editor">.
